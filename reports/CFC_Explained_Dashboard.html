The CFC outlets order-planning brain

This is a computer system that looks at how much each product sold every day across the CFC outlets (Seasons, NBH, Bistro, Gong Cha), then predicts how much each outlet will sell tomorrow — and tells the warehouse exactly how many to make. Goal: never run out, never throw away.

It is like a very experienced baker who remembers every single day of sales for 3.5 years, never forgets, and every night writes down how many of each bun to make tomorrow for every shop.

🤔The problem we solve

The CFC outlet network has two ways to lose money every single day.

Problem 1 — Stockout

Bake too little → empty shelf

Customer wants a croissant, shelf is empty, customer leaves. Lost sale, unhappy customer.

Problem 2 — Waste

Bake too much → throw away

Bread not sold by evening goes in the bin. You paid to make it; you get nothing back.

The hard part: these two pull in opposite directions. Bake more to avoid empty shelves → you waste more. Bake less to avoid waste → you run out. The right number is different for every product, every shop, every day. A human cannot compute 84 shops × 3,580 products by hand each night. The computer can.

🏗️What we built — the 9 steps

A pipeline = an assembly line. Sales data goes in one end, a daily bake-list comes out the other. Every step is finished and working (all 9 phases ✅).

Get the sales data

Pull 3.5 years of "how many sold" from the company database (Microsoft Fabric), per shop, per product, per day.

Clean & understand it

Check it's correct, find patterns: which products sell most, weekend vs weekday, festival effects.

Build "clues" (features)

For each day we give the model hints: what sold last week, the 28-day average, day-of-week, holidays, weather.

Make a simple guess (baseline)

The dumb method ("just repeat last week's average") to measure against. This is the bar to beat.

Train the smart model (LightGBM)

A machine-learning model learns the patterns and predicts demand. It beat the simple guess by ~16%.

Test it honestly (backtest)

Pretend it's 3 past months, predict, compare to what really happened. No cheating — it never sees the future.

Turn forecast into an order (newsvendor)

A forecast isn't an order. We do the maths of "cost of empty shelf vs cost of waste" to pick the smart order quantity.

Run it every night + keep learning

Nightly: produce tomorrow's bake-list. Weekly: re-train and only upgrade if the new model is actually better.

This dashboard + handoff

Show all the numbers in plain language so anyone can use it and trust it.

🎓How we train it

Two brains working together: one predicts demand, one decides the order.

Brain 1 — the forecaster (LightGBM)

LightGBM is a "gradient-boosted trees" model. Think of it as thousands of yes/no questions stacked up: Is it a weekend? Did it sell a lot last week? Is it a holiday? Is it raining? Each question refines the guess. It learned these questions automatically from 8.3 million examples.

It's 20 questions, but with thousands of questions and millions of past examples, so the final guess is very well-informed.

Important: it doesn't give one number. It gives a range — a "most-likely" (P50), a "safe" (P85) and a "very safe" (P95) level. That range is what lets us choose how cautious to be.

What do P50, P85, P95 mean? (all three, one place)

"P" = percentile = a confidence level. Think of it as a safety dial you turn up or down.

P50
Most-likely — the honest middle guess
Demand lands below this about half the time, above it the other half. Best for the cost-cheapest order. Use for thin-margin / spoils-fast items.
P85
Safe — covers a busy day
Demand stays below this 85% of days. Order to here and you'll rarely run out. A bit of waste on quiet days is the price. Good for important / steady sellers.
P95
Very safe — almost never empty
Covers 95% of days, even big spikes. Almost zero stockouts, but the most waste. Use only for must-never-miss items (and long shelf-life).
Same forecast, three caution levels. Turn the dial UP (P95) to avoid empty shelves; turn it DOWN (P50) to avoid waste. The newsvendor brain picks the right dial setting per product automatically — once it knows each product's real margin and shelf-life.

Brain 2 — the order-decider (Newsvendor)

A 100-year-old formula from running a newspaper stand: every morning decide how many papers to stock. Too few = miss sales; too many = waste. It balances:

Cost of being SHORT (Cu)the profit you lose when the shelf is empty
Cost of being OVER (Co)the money you lose throwing unsold items away

It computes a "critical ratio" = Cu ÷ (Cu + Co). High-margin, long-life items → order generously. Thin-margin, spoils-fast items → order lean. This is exactly where the real economics data plugs in.

If a product's profit is big and it doesn't spoil, make extra to be safe. If profit is small and it spoils tonight, make just enough. The formula does this product by product.

The golden rule we never broke: no cheating

When training and testing, the model is never allowed to see the future. To predict a day, it only uses data from before that day. That's why the test scores below are honest — they mirror real life, where tomorrow is genuinely unknown.

Be honest — what is the model bad at?

Rare, lumpy items (a product that sells 0, 0, 0, then 12) are genuinely hard for any model — there's little pattern to learn. These are Class C: ~5% of volume. Plan: add a specialist method (Croston/SBA) just for them. They don't hurt the money much, so it's on the backlog, not urgent.

You don't need to understand any of the maths to run it. Three routines.

Every night

Get tomorrow's bake-list

The system produces a warehouse make-list: for every product, how many units to bake and which outlets they go to, with the ₭ value. Hand it to the warehouse/production team.

pipeline predict → order picklist
Every week

Let it re-learn

It re-trains on the latest week and only swaps in the new model if it's actually better. A worse model can never go live — fully safe.

pipeline retrain
Anytime

Health check

Ask "is the world drifting away from what I learned?" It checks weather/behaviour shifts and tells you if a re-train is worth it.

pipeline monitor
When should I re-train? Only when accuracy actually drops (recent error climbs more than 10%). Weather drifting in monsoon season is normal and expected — that alone is not a reason to panic. The system tells you the difference.

♻️How it improves itself

It's not a one-time build. It's a loop that keeps getting better and guards itself against getting worse.

Champion is the model on duty

One trusted model makes today's real bake-lists.

Challenger trains in the shadows

Each week a fresh model trains on the newest data, off to the side.

They compete fairly

Both are scored on the same recent days. The challenger must win by at least 1% to earn promotion.

Only the better one is promoted

If the challenger wins, it becomes the new champion. If not, the old one stays. You can never get worse.

A drift-watcher stands guard

It watches for the world changing (new weather, new behaviour) and flags when a re-train is genuinely worth it.
