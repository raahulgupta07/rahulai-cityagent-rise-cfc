The Project

Bakery is perishable — every order is a bet

😟 Order too little

Empty shelves by noon. Lost sales, unhappy customers, wasted footfall and marketing spend.

🗑️ Order too much

Unsold buns thrown out at close. Direct money in the bin — every single day, across 98 outlets.

  • Today: outlets order daily stock from the warehouse on gut feel + last week's numbers.
  • Goal: forecast demand per product per outlet, then recommend the right order quantity automatically.
  • Why now: 4 years of clean daily sales data + the external signals are already in place.

The Order Engine

A forecast is not an order — we optimise the decision

📊 Forecast a range, not a point

The model outputs a safe band (e.g. likely 80, up to 110 on a good day) — not a single fragile number.

⚖️ Balance cost of being wrong

High-margin, cheap-to-make item → lean to order more. Costly, fast-spoiling item → order tighter. (Newsvendor logic.)

🚚 Roll up to the warehouse

Per-outlet item orders aggregate into one warehouse picklist + procurement plan, using existing supply routes.

📦 Respect real constraints

Minimum order qty, case/pack sizes, shelf-life limits all applied automatically.

Self-Learning System8 / 14
Staying Accurate Over Time

Data drift detection & guardrails

Demand patterns shift — new competitors, price changes, store remodels, economic swings. The system watches itself and flags when reality moves away from what it learned.

🔍 Drift detection

Monitors input patterns & accuracy. Alerts when a product/outlet starts behaving differently.

📉 Accuracy tracking

Forecast-vs-actual error tracked per outlet & product, daily — degradation triggers a retrain.

🛟 Safety guardrails

Sanity caps, fallback to proven baselines, human review for big swings — never blindly over/under-order.

🆕 Cold-start handling

New outlet or product with no history → category & region averages until it builds its own pattern.

🔔 Alerting

Stale data, missing feeds, accuracy drops → flagged to the team automatically.

🧪 Champion / challenger

New model versions tested against the live one before promotion — only promote if better.

Risks & Mitigations

Known risks — and how we handle them

Intermittent demand

Slow SKUs sell zero most days → specialised method (Croston), not forced into the main model.

Shelf-life data missing

Derive a proxy from product category now; request a shelf-life master for precision.

Promo data is coarse

Campaign-level only → used as signals; refine if per-SKU/outlet mapping becomes available.

Pattern shocks & drift

Festivals, weather, economy → external signals + drift monitoring + human-in-loop guardrails.
